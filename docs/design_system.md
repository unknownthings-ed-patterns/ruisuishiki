# デザインシステム（ruisuishiki）

> ビジョン声明（README/CLAUDE.md）の「世界最高峰のUI/UX」「触れた瞬間の体験」「陳腐でない」「文理問わず・初心者にも入りやすく」を実装の細部まで貫くための土台。

## 設計思想

### ① 知的な静けさを基調に、微細な愛らしさをちりばめる

Brilliant の落ち着き＋Linear のキレ＋日本の品の良い書籍デザイン（みすず書房・岩波）の余白感を混ぜる。子ども向けアプリ特有の「賑やかさ」を避ける——賑やかさは初学者には親切に見えて、すぐ陳腐になる。

### ② 余白こそが情報設計

問題1つに大胆な余白を取る。視線が問題そのものに集中できる。装飾より「白」を信頼する。

### ③ 数式・数字を一級市民として扱う

タイポグラフィの中で数字が美しいことを最優先。等幅数字（tabular-nums）で位が揃う・読みやすい。

### ④ 動きは意味を運ぶ

アニメーションは装飾ではなく、**比較を促す・気づきを支える・状態遷移を伝える**ためのもの。意味のない動きは入れない。

## カラーパレット

CSS変数で定義し、Tailwind config から参照する。ライトモード基調、ダークモード対応。

### 基調色（無彩色軸）
| トークン | ライト | ダーク | 用途 |
|---|---|---|---|
| `--background` | `oklch(0.99 0.005 90)` 紙色 | `oklch(0.16 0.01 270)` 深い藍 | ページ背景 |
| `--surface` | `oklch(0.97 0.005 90)` 余白の白 | `oklch(0.20 0.01 270)` | カード・パネル背景 |
| `--foreground` | `oklch(0.18 0.01 270)` 墨色 | `oklch(0.95 0.005 90)` 紙白 | 本文 |
| `--muted` | `oklch(0.55 0.01 270)` 薄墨 | `oklch(0.60 0.01 270)` | サブテキスト |
| `--border` | `oklch(0.90 0.005 90)` | `oklch(0.25 0.01 270)` | 罫線 |

### アクセント色（少量だけ使う）
| トークン | 色 | 用途 |
|---|---|---|
| `--accent` | `oklch(0.55 0.13 220)` 控えめな青藍 | リンク・主要ボタン |
| `--accent-warm` | `oklch(0.65 0.10 60)` 朱に寄せた橙 | 正答・気づきの瞬間 |
| `--accent-soft` | `oklch(0.85 0.05 220)` | ハイライト・ヒント表示 |
| `--success` | `oklch(0.60 0.12 145)` 抑えた緑 | 検証通過・正答 |
| `--warning` | `oklch(0.75 0.13 80)` | ヒント開示 |

### 設計ルール

- アクセント色は **1画面に1〜2色まで**。色で情報を分けすぎない
- 「正答時の温かさ」は `--accent-warm` の **背景温度の微妙な変化** で表現（点滅・派手な色変えは厳禁）
- ダークモードは「夜の図書館」のような落ち着き

## タイポグラフィ

### フォントファミリー

| 役割 | フォント | フォールバック |
|---|---|---|
| 本文（日本語） | Noto Sans JP | -apple-system, "Hiragino Sans", "游ゴシック", sans-serif |
| 見出し（日本語） | Noto Serif JP | "Hiragino Mincho ProN", "游明朝", serif |
| 数字・数式 | Inter（tabular-nums有効化） | ui-monospace, "SF Mono", monospace |
| ラテン文字本文 | Inter | -apple-system, sans-serif |

明朝の見出し＋ゴシックの本文という組み合わせは、岩波文庫の凡例や、品のある日本の書籍に通じる質感を作る。

### サイズスケール

`clamp()` を使ったレスポンシブ。子どもタブレット〜デスクトップまで同じ比率。

| トークン | サイズ | 用途 |
|---|---|---|
| `text-xs`  | clamp(11px, 0.75rem, 12px) | キャプション |
| `text-sm`  | clamp(13px, 0.875rem, 14px) | ラベル・補助 |
| `text-base`| clamp(15px, 1rem, 16px) | 本文 |
| `text-lg`  | clamp(17px, 1.125rem, 18px) | 強調本文 |
| `text-xl`  | clamp(19px, 1.25rem, 20px) | 小見出し |
| `text-2xl` | clamp(22px, 1.5rem, 24px) | 見出し |
| `text-3xl` | clamp(28px, 1.875rem, 32px) | セクション見出し |
| `text-4xl` | clamp(36px, 2.5rem, 44px) | ファーストビュー |
| `text-5xl` | clamp(48px, 3.5rem, 60px) | ヒーロー |

### 行間・字間

- 日本語本文：`line-height: 1.8` `letter-spacing: 0.04em`（読書品質）
- 数字・式：`line-height: 1.4` `font-feature-settings: "tnum"` （tabular-nums）
- 見出し（明朝）：`line-height: 1.4` `letter-spacing: 0.06em`（明朝の品を出す）

## コンポーネントの基本原則

### 境界線より影より色面

- ボーダーは **1px solid var(--border)** のみ（複雑な影は使わない）
- カードの分離は背景色の濃度差で行う
- フォーカスリングは `--accent` を 2px、外側 offset 2px

### ボタン

| 種別 | デザイン |
|---|---|
| Primary | `--accent` 背景＋紙白文字。角丸 8px |
| Secondary | 透明背景＋`--accent` ボーダー＋`--accent` 文字 |
| Ghost | 完全に透明＋ホバーで `--surface` |
| Quiet | 文字色の変化のみ |

ボタンは小さくしすぎない——タッチ操作（子どもタブレット）対応のため、最小 44×44px。

## アニメーション原則

framer-motion を使う。下記の指針：

### 動きの「重さ」を統一する

- `--ease-smooth`: `cubic-bezier(0.32, 0.72, 0, 1)`（Apple流の余韻）
- `--ease-precise`: `cubic-bezier(0.4, 0, 0.2, 1)`（マテリアル流）
- `--duration-fast`: 150ms（ボタン押下フィードバック）
- `--duration-base`: 300ms（コンテンツ遷移）
- `--duration-slow`: 500ms（ページ遷移・「比較指さし」での前題スライドイン）

### 主要シーン

1. **「比較指さし」ヒント開示**：前題が左から `x: -16 → 0`、`opacity: 0 → 1`、500ms `ease-smooth`
2. **正答時**：背景が `--accent-warm` を一瞬まとう（500ms かけて温度上昇→冷却）。派手な色変化なし
3. **ヒント階層 1 → 2 → 3 へのフェードイン**：300ms。前の階層は消えず、下に積層
4. **問題遷移**：上方向に 24px スライド＋フェード（次へ）/ 下方向（前へ）
5. **ボタンホバー**：transform: scale(1.02)、150ms。クリック時 0.98。

### アクセシビリティ

- `prefers-reduced-motion: reduce` を尊重——アニメーション距離をゼロに（透明度のみ）

## アイコン

- **lucide-react**（shadcn/ui標準）を主に使用
- 線幅 1.5px で統一（細めの線が「知的な静けさ」と合う）
- カスタムアイコンは SVG で書き、`currentColor` でテーマと連動

---

## アクセシビリティ

- WCAG AA 準拠を目標（コントラスト比4.5:1以上）
- すべての操作がキーボードで完結する（Tab・Enter・Space・Esc）
- フォーカス可視化を強く（`--accent` のリング）
- 色だけで情報を伝えない（アイコン or テキスト併記）
- スクリーンリーダー用のラベル必須
- フォントサイズはユーザー設定を尊重（rem 基準）
- 特別支援対応：行間広め・余白多め・コントラスト調整可能

## トーンと文体

UI上の日本語の文体は以下：

- **学習者ビュー**：です・ます調。子どもにも読める平易な漢字とひらがな
- **教師ビュー**：簡潔・専門用語可（文型・解式・変化オペレータ）
- **エラー・警告**：原因と「次の一手」を1文で。否定表現を避ける
- **称賛**：「すごい！」より「気づけたね」「自分で見つけられたね」——成長マインドセットの言葉

## 設計の参考

このシステムが「陳腐でない」を貫けているかを判断する基準：

- **Brilliant** の落ち着いた青と余白感
- **Linear** の精緻なインタラクション・タイポ
- **Notion** の白の使い方
- **岩波文庫**・**みすず書房** の明朝見出しと本文の対比
- **Apple** のアニメーションの「重さ」と余韻

陳腐への警告サイン：
- 派手な原色グラデーション
- 過剰なシャドウ・ガラスモーフィズム
- キャラクター・絵文字で**代替する**愛らしさ（微細な動きで支える愛らしさは可）
- **内在的動機を損なう**外付けのバッジ・ポイント乱発（次節で詳述）

これらが入ってきたら、立ち止まる。

## ゲーミフィケーション設計の指針

ゲーミフィケーションは将来検討する。ただし、本プロジェクトでは「内在的」と「外付け」を厳密に区別する。

### 内在的ゲーミフィケーション（戸田の推理式に最初から組み込まれている）

積極的に実装する。陳腐にならない。

- **「喜びのグレードアップ」の可視化**：質的変化問題に到達したときに「**同じ構造を発見した瞬間**」を温度感のあるアニメーションで支える
- **系列の達成感**：その一→その二→その三の進行を、線分や航路のメタファで visual に
- **作問が採用される実績**：子どもが作った問題が文型検査を通り、他の学習者に出題される——これ自体が大きな承認
- **つまずきオペレータの解消ツリー**：自分が弱かった「逆」「質的変化」が時間とともに解けていく軌跡を visual に
- **自分の文型カタログ**：解いた・作った文型が増えていく図書館のような構成

### 外付けゲーミフィケーション（慎重に・最小限・上品に）

検討対象だが、入れるとしても以下の制約：

- バッジは控えめなアイコン（星★ではなく、版画調の小さなスタンプなど）
- ポイント・コインは原則使わない（学習を計量化することの教育的副作用）
- 連続日数の表示は**ローカルストレージのみ**、SNS共有を煽らない
- キャラクターは導入しない（あえて）

判断軸：「これは戸田の哲学（自分で考えること自体に価値）を強化するか、それとも外発的動機に置き換えるか」

### 第3弾仕様書との整合

将来 MVPの段階7（拡張フェーズ）で内在的ゲーミフィケーションを本格実装する。Codex に委ねる機械作業の比率を上げる際に、視覚言語の質はメインの Claude セッションで守る。
