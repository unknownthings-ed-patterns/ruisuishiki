# 解答入力・判定系の拡張 設計

作成: Opus 4.8（2026-07-01）。**本来は Fable が設計する領分だが、セッション時点で Fable 5 が利用不可のため、メイン（Opus）が設計。Fable 復帰時にレビューを受ける前提。**

## なぜ要るか（背景）

三角関数ユニットを設計（`docs/trig_series_design_fable.md`）通り実装するには、現行の解答入力系では表現できない 3 つの要求がある：

1. **根号値**：sin60°=√3/2、cos45°=1/√2、tan60°=√3（系列1の古典的特別角・系列2でも頻出）
2. **π を含むラジアン角**：π/6、5π/6、3π/4（系列2・3 の答えそのもの）
3. **1 問に複数解**：三角方程式 sinθ=1/2 → θ=π/6 **かつ** 5π/6（系列3。`answer: number` 単数では持てない）

現状（`frontend/src/app/learn/play/page.tsx` の `parseAnswer` 53行〜、判定 247行・`Math.abs(parsed - step.answer) < 1e-6`／`types.ts` の `LearnerStep.answer: number`）は **分数 `a/b` と小数のみ**。系列1は無理数を避けて有理数角だけに縮めて実装済み（`seriesTrig.ts` 冒頭コメント参照）。

## 絶対制約（土台）

- **学習者ビューはバックエンドなし＝静的配信**（`CLAUDE.md` ビジョン声明・課金モデル）。**判定は client-side JS で完結**。**SymPy はサーバ判定に使えない**（生成時のみ）。
  → だから **記号代数の厳密同値判定でなく、数値評価＋許容誤差**が現実的な軸。√3/2→0.8660254、π/6→0.5235988 に評価して比較する。これで根号値も π 角も**同じ仕組みで**扱える（最大のレバレッジ）。
- **既存全系列（`answer: number`）を壊さない**——後方互換の加法的拡張。
- **ローフロア／世界最高峰**——入力UIは子ども・文系の大人も迷わない。
- **「比較を指さす」ヒント哲学**と衝突しない（判定系の話なので中立）。

## 設計

### 1. 入力構文 — 安全な式評価器（`eval()` は使わない）

`parseAnswer` を、小さな**再帰下降パーサ**（トークナイザ＋shunting-yard でも可）に置き換える。受け付けるトークン：

| 種類 | 書き方 | 評価 |
|---|---|---|
| 整数・小数 | `3`, `0.5`, `-2` | そのまま（既存維持） |
| 分数 | `a/b` | `a÷b`（既存維持） |
| π | `π` または `pi` | `Math.PI` |
| 根号 | `√3`, `√(3)`, `sqrt(3)` | `Math.sqrt(3)` |
| 演算 | `+ - * /`・括弧 | 通常の優先順位 |
| 暗黙の積 | `5π`, `2√3` | `5*π`, `2*√3`（ローフロア配慮） |

例：`√3/2`、`-1/√2`、`pi/6`、`5pi/6`、`2π/3`。**記号（π・√）と英単語（pi・sqrt）の両方**を受ける。

### 2. 判定 — `evaluateAnswer(input): number | null`

- 既存の整数・小数・`a/b` は**まったく同じ結果**を返す（後方互換の要）。
- π・√・括弧・暗黙積を含む式を **float に評価**して返す。パース失敗は `null`。
- 比較は既存どおり `Math.abs(評価値 - 期待値) < TOL`。**期待値はコード側でも同じ評価系で算出**（`Math.sqrt(3)/2` 等）するので、学習者が記号形で正しく入れれば**厳密に一致**する。

### 3. データモデル — 複数解（`types.ts` 加法的拡張）

```ts
export type LearnerStep = {
  // ...既存はそのまま...
  answer: number;                 // 既存（単数）維持
  /** 複数解を全部答えさせる問題（三角方程式等）。指定時は judging が answer より優先。
   *  学習者は "," 区切りで全解を入力。順不同・許容誤差で多重集合一致。 */
  solutionSet?: number[];
};
```

- `solutionSet` 指定時：学習者入力を `,`（と空白）で分割→各片を `evaluateAnswer`→**多重集合一致**（期待集合の各要素がちょうど 1 回ずつ・過不足なし・許容誤差）で判定。
- 既存系列は `solutionSet` 不在 → 従来通り `answer` 単数判定。**完全後方互換**。
- 単数だが複数の許容形（例「30° でも π/6 でも可」）は当面不要（数値評価が `1/2`・`0.5` 等の形の揺れは吸収する）。必要なら後日 `acceptedAnswers?: number[]`（any-of）を別途追加。

### 4. 入力UI（`play/page.tsx`）

- 既存の単一テキスト入力を維持。**π・√ を挿入する小さなパレットボタン**をカーソル位置に挿入。
- パレット・記法ヘルプは**その step が π/√/複数解を使うときだけ表示**する（既存の素の問題を汚さない＝ローフロア維持）。そのため step に軽いフラグを足す：
  ```ts
  /** 入力補助の表示制御（無指定＝従来の素の数値入力）。 */
  inputAffordances?: ("pi" | "sqrt" | "multi")[];
  ```
- 複数解 step は入力欄下に小さく：「解が複数あるときは "," で区切って全部入れてね（例: π/6, 5π/6）」。
- π/√ step は：「√は √ ボタンか sqrt、πは π ボタンか pi。例: √3/2, pi/6」。

### 5. 既存系列への影響

加法的。`evaluateAnswer` は整数・小数・`a/b` で従来と同一出力（移行時に既存全系列の answer 値を 1 度評価して回帰確認）。`solutionSet`・`inputAffordances` は optional。**既存系列のデータ変更は不要**。

### 6. 系列1への波及（別タスク・方針のみ）

入力系拡張後、系列1を Fable 設計表の**元の角**（sin45°=1/√2、sin60°=√3/2、tan60°=√3 等）に戻し、有理数縛りを解くことを**推奨**。胚細胞・オペレータ列は不変のまま値だけ差し替え。系列1が恒久的に有理数限定にならないようにする。実装は入力系が乗ってから小タスクで。

## 判断ずみ（岩井先生・2026-07-01）

1. **判定の厳しさ → 厳密（記号入力を促す）で確定**。誤差 `TOL = 1e-6`。丸めた小数（√3/2 を `0.87`・`0.866`）は不正解。記号形（√3/2・pi/6）で正確に表せたときだけ正解。「値を正確に表す」理解を育てる・誤判定ゼロ。
2. **複数解の入力UX → `,` 区切りの単一入力欄で確定**。順不同・多重集合一致。入力欄下に例示「解が複数あるときは "," で区切って全部入れてね（例: π/6, 5π/6）」。複数の入力ボックスは不採用（個数がヒントになる・UIが重い）。

### 未確定（系列3 実装時に詰める・入力系実装はブロックしない）

3. **一般解（+2nπ）**：系列3 step1 は一般解 `π/6 + 2nπ` 形。数値判定では無限集合を扱えない。
   - (推奨) 系列3 は**範囲付き**（0≤θ<2π 等で有限集合）に寄せ、一般解は derivation/もっと深く で言葉として扱う。
   - 系列3 の実装時に詰める。
